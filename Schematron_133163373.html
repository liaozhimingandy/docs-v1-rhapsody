Schematron

The Schematron filter uses a Schematron file (*.sch) to validate XML documents.

Schematron is a language and toolkit for making assertions about patterns found in XML documents. Because it uses paths, rather than grammars, it can be used to assert many constraints that cannot be expressed by DTDs or XML schemas.

It differs in basic concept from other schema languages in that it is based on finding tree patterns in the parsed document. This approach allows many kinds of structures to be represented, which are inconvenient and difficult in grammar-based schema languages. For example, Schematron is useful for validating the complex rules associated with HL7 Continuity of Care Document (CCD) and Clinical Document Architecture (CDA) documents.

Schematron allows you to develop and mix two kinds of schemas:

  • Report elements allow you to diagnose which variant of a language you are dealing with.
  • Assert elements allow you to confirm that the document conforms to a particular schema.

The Schematron filter only supports iso schematron formats. Any attempt to use other formats, for example schematron 1.5 will result in a misleading XML parsing failure error.

Configuration Properties

Property

Description

Maximum Concurrency

The maximum level of concurrency for this filter. Limiting the level of concurrency limits the memory usage. 0 means unlimited. Refer to Maximum Concurrency for details.

Schematron Definition (Zip Archive)

The Schematron definition against which the input must be validated. The definition must be stored in a zipped archive.

On Validation Success

The action to take if validation succeeds:

  • Continue processing on route (default) - outputs the original message.
  • Output SVRL report - outputs the SVRL report.

On Validation Failure

The action to take if validation fails. Default: Output SVRL report

  • Output SVRL report - outputs the SVRL report.
  • Send to error connector - outputs the original message via the Error connector.
  • Continue processing on route - outputs the original message.

Schematron Phase

The Schematron phase to process. If not present or empty, then all phases are processed.
