Using Rhapsody Documentation

Rhapsody product documentation can be accessed through the Rhapsody Documentation Portal (Doki). It intended as a reference for users who have undergone Rhapsody training.

It is recommended that you use a high-resolution display to view Doki pages. 

Rhapsody Documentation Overview

Rhapsody product documentation envisages three types of users for Rhapsody:

  • Monitor - users who monitor the day-to-day health of the engine, such as: logging into the Management Console; viewing and managing messages; and performing tasks that allow them to monitor the day-to-day health of the engine.
  • Developer - users who design and develop routes and interfaces for the engine, such as: logging into Rhapsody IDE; viewing and managing configurations; and developing interfaces (as well as tasks allotted to the monitor).
  • Administrator - users who perform administrative tasks on the engine, such as: managing users and access rights; and maintaining backup schedules and licenses (as well as tasks allotted to the monitor and developer).

The documentation is orientated towards these three overlapping roles and is roughly categorized as such.

Rhapsody products are supplied with a full set of administration and installation manuals, in both PDF and web-based formats. The Printable Documents page provides links to downloadable versions of the manuals, release notes and known issues in PDF format for each version of Rhapsody.
