register and registration-prompt

This section describes how to register the Rhapsody engine.

Alternatives

You can register in one of the following ways when using a script:

  1. Interactive registration (<registration-prompt/>).
  2. Automated registration with automated FTP delivery, but falls back to interactive delivery if FTP fails (<interactive-delivery/>).
  3. Automated registration with deferred manual delivery. This writes the registration file to a directory and you can manually email the registrations through later.
  4. Automated registration with fully automated delivery. FTP or email delivery scripted fallback options.

Example

<!-- If FTP fails, Rhapsody tries to email the registration information. If email fails as well, then Rhapsody saves
the registration information to the directory specified in manual dir -->
<register>
    <organization>Acme Corporation</organization>
    <address>
        <line1>181 Grafton Road</line1>
        <line2>Grafton</line2>
        <city>Auckland</city>
        <postcode>1010</postcode>
        <state>Auckland</state>
        <country>New Zealand</country>
    </address>
    <engine>
        <name>Rhapsody</name>
        <purpose>Experimental</purpose>
    </engine>

    <delivery>
        <ftp/>
        <fallback>
            <email/>
            <manual dir="C:\registrations"/>
        </fallback>
    </delivery>
</register>

<!-- Don't try to deliver the registration, instead write the registration information to a folder that can be sent
to Rhapsody Global later -->
  <register>
    <organization>Acme Corporation</organization>
    <address>
        <line1>181 Grafton Road</line1>
        <line2>Grafton</line2>
        <city>Auckland</city>
        <postcode>1010</postcode>
        <state>Auckland</state>
        <country>New Zealand</country>
    </address>
    <engine>
        <name>Rhapsody</name>
        <purpose>Experimental</purpose>
    </engine>

    <delivery>
        <manual dir="C:/registrations"/>
    </delivery>
</register>

<!-- Register, but if the FTP registration fails display a dialog and ask the user how to deliver the registration -->
<register>
    <organization>Acme Corporation</organization>
    <address>
        <line1>181 Grafton Road</line1>
        <line2>Mt Eden</line2>
        <city>Grafton</city>
        <postcode>1010</postcode>
        <state>Auckland</state>
        <country>New Zealand</country>
    </address>
    <engine>
        <name>Rhapsody</name>
        <purpose>Experimental</purpose>
    </engine>

    <interactive-delivery/>
</register>

<!-- Full manual registration -->
<registration-prompt/>
