Running the Rhapsody on *nix

Use the following commands to start, restart or stop the Rhapsody service/daemon on *nix environments:

Enter the command

To

./rhapsody.sh start

Start the Rhapsody engine.

./rhapsody.sh restart

Restart the Rhapsody engine.

./rhapsody.sh stop

Stop the Rhapsody engine.

./rhapsody.sh status

Check the status of the Rhapsody engine.

Starting Rhapsody Automatically on *nix Environments

Complete the following steps as the superuser or root.

The Rhapsody startup script (rhapsody.sh) contains references to the relative path of the wrapper tools. To incorporate this file into the boot time startup, you must create a link to the script from the system's startup directory. For example, on most LinuxÂ® systems, this is the /etc/init.d directory.

To create the link:

  1. If the install directory is: /<directory>/rhapsody/rhapsody-engine-6, create the link by entering the following command: 

    ln -s /<directory>/rhapsody/rhapsody-engine-6/bin/rhapsody.sh /etc/init.d/rhapsody

  2. Incorporate the Rhapsody script into the startup process, as appropriate for the system.

    If the script rhapsody.sh is copied to the /etc/init.d directory, the paths will be incorrect and the wrapper tools will not be available.
