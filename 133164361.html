Uninstalling the Rhapsody Engine on *nix

The process for uninstalling the Rhapsody™ engine is the same for all *nix variants.

These instructions have been written using Linux® as an example.

The uninstallation process can be run in console mode or with a GUI. You must have administrative privileges to uninstall on a *nix-based system. If you do not have administrative privileges, the following error is displayed:

This installation can only be uninstalled by a user with administrative privileges.

To uninstall the Rhapsody Engine on a *nix environment:

  1. Navigate to the directory where the uninstall script is located. For example, if you installed the Rhapsody engine in the default directory, enter:

    cd /opt/orionhealth/rhapsody

  2. Run the script in the console mode using the -c argument.

    ./uninstall -c

  3. The following message is displayed:

    Are you sure you want to completely remove Rhapsody <majorVersion>.<minorVersion>.<servicePack> and all of its components?
Yes [y, Enter], No [n]

    Press ENTER to confirm that you want to uninstall the Rhapsody engine.

  4. The following message is displayed:

    Selecting these options will permanently delete the associated data or logs that were created while using Rhapsody.
Delete Rhapsody Data?
Yes [y], No [n, Enter]

    Press ENTER to confirm that you want to delete all your Rhapsody data.

    This also deletes the configuration data.

  5. The following message is displayed:

    Delete Log Files?
Yes [y], No [n, Enter]

    Press ENTER to confirm that you want to delete the Rhapsody engine logs.

  6. When the wizard finishes uninstalling Rhapsody, the following message is displayed:

    Rhapsody <majorVersion>.<minorVersion>.<servicePack> was successfully removed from your computer.

    Any files or directories that could not be removed are displayed. You may delete them manually, if required.
