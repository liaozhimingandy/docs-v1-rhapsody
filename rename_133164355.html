rename

This section describes how to rename folders, communication points, and routes. It is intended to be used to rename a template interface after loading it from an RLC.

The following elements are available:

  • <rename-locker/>
  • <rename-folder/>
  • <rename-communication-point/>
  • <rename-route/>
  • Folders can be moved to other locations within the same locker.
  • Lockers can be renamed, but cannot be moved.
  • Communication points and routes can only be renamed within their current folder, but cannot be moved to another folder.

Example

<rename-locker old="HospitalA" new="HospitalB"/>
<rename-folder old="Test/A/B/C" new="Test/Observations"/>
<rename-communication-point folder="HospitalB/Test/Observations" old="Input 22" new="Observation Input"/>
<rename-communication-point folder="HospitalB/Test/Observations" old="Output 22" new="Observation Output"/>
<rename-route folder="HospitalB/Test/Observations" old="Route 22" new="Observation Test Route"/>

The folder path needs to include the locker.
