Using SoapUI

SoapUI is a Java-based tool for testing Web services. A free version can be downloaded from http://www.soapui.org/. It has some built-in support for WS-Security, WS-Addressing and MTOM. It supports the concept of projects, where each project is associated with a WSDL.

New projects are created by providing the URL or filename of the WSDL.

As all services within the Rhapsody Web Services API support both the SOAP 1.1 and 1.2 bindings, SoapUI will create two separate interfaces for each imported project â€” one for each SOAP binding. By default, it creates sample requests for each operation to allow them to be called very easily.

The template can then be manually populated to build up a request.

Authentication

Authentication is performed by clicking the Aut button in the bottom left corner of the request, and then providing a username and password combination with permission to call the Web service. If the <soap:Header/> element in the request is then right-clicked, the displayed context menu allows a WSS Username Token to be inserted. Select this option, and choose the PasswordText option from the list of password types.

Finally, the Web service request is run by clicking the green go arrow in the top-left corner. This sends the SOAP request to Rhapsody and then displays the response it receives.

WS-Addressing

WS-Addressing is required for some calls to the Web Services API. To configure WS-Addressing, click the WS-A button at the bottom of the request. Each of the WS-Addressing elements can then be configured to any value.

Note that although SoapUI often sets the default action correctly, it performs no validation on any of the values provided. If they are set incorrectly, it is likely that an error will occur. For example, if WS-Addressing is required by a call to the API, the Randomly generate MessageId checkbox (that is unchecked by default) must be checked in order for it to be accepted by Rhapsody.

The Reply To setting is used to configure the wsa:ReplyTo header. This should not be used when calling the Rhapsody Web Services API, as at present all of these services must be called synchronously.

MTOM

The Message Transmission Optimization Mechanism (MTOM) is a mechanism to providing binary attachments to a SOAP message. When this is used, the HTTP request or response contains a MIME multi-part message with the first part containing the SOAP message, and the remainder containing the attachments. These attachments have identifiers allowing them to be referenced by the SOAP message itself. Many client tools allow MTOM to be used relatively transparently.

Currently only a single operation in the Rhapsody Web Services API makes use of MTOM. This is the getMessage operation in the Message Searching service. If the message body is requested, then it is returned using MTOM in the response. No requests in the Rhapsody Web Services API require a parameter to be provided using MTOM.

Using MTOM for SOAP responses within SoapUI is very simple. Run the request normally and if an MTOM attachment is available it will be indicated at the bottom of the response as Attachments (1).

Clicking the Attachments button shows the available attachments and allows them to be opened or saved to disk.
