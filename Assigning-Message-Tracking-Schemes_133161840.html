Assigning Message Tracking Schemes

The Message Tracking tab in the communication point properties dialog is used to assign message tracking schemes to communication points, for example:



The following options are available on this dialog tab:

Option

Description

Enable Message Tracking for this Communication Point checkbox

By default, message tracking is not enabled on any communication point. This checkbox must be selected before any message tracking options are available.

Scheme drop-down list

This drop-down list allows the appropriate message tracking scheme to be selected for use on this communication point.

View button

Enables the message tracking scheme to be viewed in read-only mode.

Override button

Enables parts of the message tracking scheme to be overridden for this particular communication point only.

New button

Creates a new message tracking scheme to be used with this communication point through the Message Tracking Scheme manager.

Scheme Manager button

Launches the Message Tracking Scheme manager so that all available message tracking schemes can be viewed and edited.
Summary window Displays a summary of the selected message tracking scheme.

Overriding Message Tracking Schemes

There are times when Rhapsody is connecting to multiple other systems and they all use the same settings for message tracking, except for one system that behaves slightly differently. For example, one remote system returns a different acknowledgment code to indicate a failure.

A separate message tracking scheme for each system would resolve this problem, but that is often excessive when only a single modification needs to be made. Instead, the message tracking scheme may be overridden by the communication point. When message tracking schemes are overridden by a communication, the overridden values take precedence over any values defined on the scheme itself.

Not all configuration options in the message tracking scheme are available to be overridden by a particular communication point: only those options relating to the behavior after a message is matched may be overridden (that is, everything on the Response Handling and Error Handling tabs of the Message Tracking Scheme dialog).
