De-batching a Message Set

This scenario involves reading a batched message set that is separated by the non-printable characters: ETX and EOT (End of Text and End Of Transmission respectively). Then splitting this message into several messages whenever this separator pair occurs.

Configuring the Filter

Debatching a message set that contains non-printable characters, requires configuring two parameters:

  1. The Mode must be set to De-batch.
  2. The Message Separator Expression requires a regular expression string that identifies the separator characters.

For separating ETX and EOT we need to refer to an ascii table to determine the appropriate hexadecimal characters. For ETX and EOT the hexadecimal values are 03 and 04 respectively. Therefore, the regular expression string is \x03\0x04.

Alternatively we can define a unicode escape sequence which would look like \u0003\0u0004. These escape sequences are not case sensitive and can be defined in lower or upper case.
