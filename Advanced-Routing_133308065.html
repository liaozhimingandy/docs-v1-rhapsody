Advanced Routing

Advanced Routing enables a communication point to re-route messages, either upon successful message transmission or failed message transmission, so that further actions can be performed on these messages:

  • Continued Routing on Success
  • Continued Routing on Failure

Before the advent of Advanced Routing, it was not possible to continue processing messages on the route upon successful transmission; messages that failed to transmit could only be sent to the Error Queue, requiring manual user intervention.

A typical use case involves Rhapsody uploading messages to an external server and based on the response code, either re-routing a message if it fails to upload or notifying users (by email, for example) if it uploads successfully.
