Toolbar

Rhapsody IDE provides a toolbar and keyboard shortcuts to assist you in navigating the application:

Use the toolbar buttons to:

  • Quickly access commonly used functions.
  • Perform general administrative functions.
  • Navigate within Rhapsody IDE.

Toolbar buttons that are not available for selection are displayed in grey. For example, if you have not checked out a component, the Check-in toolbar button is disabled and cannot be selected.

The toolbar is displayed by default. To hide the toolbar display, uncheck the Standard command from the Toolbars submenu of the View menu.

You can reposition the toolbar vertically or horizontally, or float the toolbar by clicking on the grip bar and dragging the toolbar to the required position.

The function of the toolbar buttons are listed in the following table:

Save the current configuration to simplify the task of moving a Rhapsody configuration from one machine to another. A Rhapsody configuration file contains the working details for all routes, communication points, and filters, along with any associated message and mapper definition files, XML configuration files, custom modules, security certificates and third-party drivers.


Reload a saved configuration.


Check in all routes and communication points.

Ctrl+H

Check out the currently selected route or communication point. This action locks the selected component for editing by the current user only; other users will be able to view but not change its configuration until the component is checked in again.

Checking out a selected route or communication point does not stop it. This means that Rhapsody is able to continue message processing using the 'old' configuration until any changes are checked in.

Ctrl+G

Check in the currently selected route or communication point. This action briefly stops and restarts the selected component.

Ctrl+J

Cancel a check out of either a selected component or the entire Rhapsody engine, and reverts the configuration back to its original state. This action must be confirmed.

Following an 'Undo Check Out', all previously checked out items will show as checked in.
