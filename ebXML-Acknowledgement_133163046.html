ebXML Acknowledgement

The ebXML Acknowledgment filter generates acknowledgment messages for received messages which have been parsed by the ebXML Filter and have been found to contain an AckRequested element.

Configuration of the ebXML Acknowledgment Filter is limited to deciding whether to sign the acknowledgment messages if requested by the sender of the original messages. Signing can be disabled, regardless of whether a signed acknowledgment was requested by the sender of the original message.

It is recommended that, as a best practice, all receiving routes implementing ebXML functionality include either the ebXML Acknowledgment filter or the ebXML AutoReply filter, to ensure that any messages received with an AckRequested element are handled correctly. Refer to ebXML Filters and ebXML AutoReply for details.

Configuration Properties

Property

Description

Message Signing

Whether the filter should sign generated acknowledgment messages if the sender of the original message has requested it:

  • Do Not Sign Messages (default).

  • As Per Message Property - use the message property eb:Sign with a value of true or false. If message signing should occur on a message by message basis, set eb:Sign to true to signify that an acknowledgment should be signed.

  • Sign Messages.

Signing Algorithm

The algorithm to use to sign messages. Currently RSA with SHA-1 and DSA with SHA-1 are supported. The alternative message property is eb:SignatureAlgorithm.

  • rsa-sha1 (default).
  • dsa-sha1.

Signing Key

The private key to use to sign messages. The alternative message property is eb:SignatureKeyAlias, whose value must be the alias of the private key to use.
