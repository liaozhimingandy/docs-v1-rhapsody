How Bouncy Castle Handles Rijndael Keys

Keys saved using the Rijndael algorithm are actually saved with the Advanced Encryption Standard (AES) algorithm in certain cases.

From the Symmetric Keys tab in the Certificate and Key Manager, generate:

  1. A Rijndael key using the default key size for the algorithm. The key is generated with the AES algorithm and a key size of 128 bits.
  2. A Rijndael key of size 192 bits. The key is generated with the AES algorithm and a key size of 192 bits.
  3. A Rijndael key of size 256 bits. The key is generated with the AES algorithm and a key size of 256 bits.

  4. A Rijndael key of size 255 bits. The key is generated with the Rijndael algorithm and a key size of 256 bits.

    255 bits is a valid size for the AES algorithm, but Bouncy Castle creates the key using the Rijndael algorithm.

  5. A Rijndael key of size 168 bits. The key is generated with the Rijndael algorithm and a key size of 168 bits. 



The Advanced Encryption Standard (AES) is a subset of the Rijndael algorithm. The AES algorithm is a 128-bit block cipher that supports three different key sizes: 128, 192 and 256 bits. The Rijndael algorithm has a greater range of valid key sizes. Bouncy Castle checks the key size for the Rijndael algorithm and if it meets the AES standard, then it saves the key as AES.
