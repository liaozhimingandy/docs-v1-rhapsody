Administrative Control of Rhapsody Server Authentication

Rhapsody allows the system administrator limited control over how the Rhapsody server authentication mechanism works within the Rhapsody IDE, that is to enforce the use of certificates issued by a trusted certificate authority rather than self-signed certificates.

Specifically, the system administrator can:

  • Prevent a non-administrative user from accepting a self-signed certificate when connecting to a Rhapsody engine.
  • Pre-configure a Rhapsody IDE computer with certificate details for any number of Rhapsody engines so that a non-administrative user can connect to them without needing to determine if they can trust the certificate themselves.
