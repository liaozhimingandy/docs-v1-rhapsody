Timer

The Timer communication point is used to trigger an event in the Rhapsody engine. At a given interval, the communication point sends a copy of a static file for processing. This file does not usually contain a message and is typically used to trigger an event in a route; for example, the extraction of the message using the database filter.

The communication point sends its first message as soon as it has started and subsequent messages thereafter according to its refresh rate.

Supported Operational Modes: Input

Input Mode Properties

Property

Description

Body Template File

The static content of the body template file is used as the message content every time the communication point triggers. Select the Browse button to search and attach the file to be used.

Refresh Rate

The refresh rate indicates the time (in milliseconds) between new messages that are created by this communication point.

The minimum refresh rate supported by the communication point in practice is 1,000 milliseconds.
