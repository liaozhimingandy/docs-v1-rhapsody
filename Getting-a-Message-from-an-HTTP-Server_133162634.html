Getting a Message from an HTTP Server

The HTTP Client communication point enables your Rhapsody engine to get information from or send information to an HTTP server. With web services becoming more predominant this communication point is becoming more popular in today's healthcare enterprises.

For reference material about this communication point, refer to:

  • HTTP Client

In this scenario, an HTTP Client communication point is configured to use the HTTP GET method to retrieve a message from an HTTP server and put the retrieved HTML into a directory:

  1. Create an HTTP Client communication point in Input mode and enter http://www.orionhealth.com in the URL property, as shown in the following screenshot:

  2. Create a Directory communication point and enter c:\temp\out in the Output Directory property. This is where the HTML messages retrieved from the HTTP Client are sent.

     

  3. Create a new route and add the communication points to the routes. Connect the two communication points and then check in the route and the communication points.
  4. A file is created in the Output Directory, every minute, based on the date and time. This is because the HTTP Client communication point, by default, connects to the server every minute and doesn't check to see that the page has changed.
  5. To send a message when the page is changed, change the Set If-Modified-Since header property to True. The communication point will then check the Last-Modified HTTP header and only place the message in the Rhapsody route if it is later than the value in the previous GET call.
