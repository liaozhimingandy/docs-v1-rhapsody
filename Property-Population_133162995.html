Property Population

The Property Population filter is used to insert information into message properties from fields in the message or from values entered during configuration.

Configuration Properties

Property

Description

Literal -> Property Mapping

Used to configure message properties that will be set to values. Click the Browse button to display the Literal -> Property Mapping dialog. Enter or select a property (in the Property column) and enter the required value for this property to be set to (in the Literal column).

Definition File

Identifies the Message Definition describing the layout of the message. Only required if you are setting properties from fields in the message.

Field -> Property Mapping

The Field -> Property Mapping field is only available if a Definition File has been specified. Click the Browse button to display the Field-> Property Mapping dialog. Enter or select a property (in the Property column) and enter the required value for this property to be set to (in the Source Field column).

On Field Missing

The action to be performed if the message does not contain a field that is mapped to a property:

  • Continue processing on route - the property is not set and the message continues on the route.
  • Send to error connector - the message is sent to the error connector.

Only required if Field->Property Mappings is configured.
