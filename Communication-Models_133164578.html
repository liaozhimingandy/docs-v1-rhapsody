Communication Models

Rhapsody is a high-performance messaging system capable of handling millions of messages per day on standard hardware configurations. It supports a wide range of connection methods and communications protocols and provides an extensive set of tools which allow business logic to be applied to messages as they traverse the engine.

The integrity of messages is maintained by a high-performance data store to which messages are persisted (stored) whenever a change is made to a message. The combination of business logic and message persistence allows Rhapsody to guarantee the delivery of messages.

The engine supports a number of communication models on a path-by-path basis, including:

  • Store-and-Forward
  • Pass-through

A single instance of the Rhapsody engine is able to support multiple communication models, as required by the endpoints with which it communicates.

Examples of Implementation

HL7 Version Management - systems from different vendors often handle different versions of HL7 or custom variants of the same HL7 version. Rhapsody can accept and transform the input version to the version required by the downstream systems.

Communications Management - in situations where a downstream system may be erratic in its response and impacting the ability of an upstream system to meet its performance requirements, Rhapsody can accept the input messages and manage the transmission process to the downstream system. When communications protocols supported by upstream and downstream systems are incompatible, Rhapsody can arbitrate between the systems by managing communications with each system appropriately.

Format Transformation - Rhapsody supports the complete transformation of a message from one format to another. For example, a sending system may transmit messages in XML (Extensible Markup Language) format, but the receiving system requires the message to be in an HL7 version.
