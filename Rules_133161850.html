Rules

The Rules tab in the X12 Project Editor provides the ability to enable or disable semantic validation rules (SNIP Levels 3-5), and provides searchable metadata for each specific rule.

When disabling rules:

  • Validation reports generated by the X12 Validator filter using a custom X12 project will not contain errors for any disabled rules.
  • When the X12 Validator filter generates a 999 acknowledgment, the acknowledgment will not have respective IK3 and IK4 segments. As expected, this has a flow-on effect to the IK5 and AK9 segments.

The Rules tab displays the following information:

Column
Description
ID The rule number.
Transaction Set The transaction sets the rule applies to.
Loop The loops the rule applies to.
Tag The segment tag the rule applies to.
Description A description of the rule.
Error Code The error code generated by the rule if it fails.
Enabled Whether the rule is enabled or disabled.

You can perform the following actions through the Rules tab:

Action
Description
Enable/Disable Select the Enabled checkbox to enable the associated rule or uncheck it to disable the rule.
Filter Filter the displayed rules using a text-based filter, or by transaction set.
