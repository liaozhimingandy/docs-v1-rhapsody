Modifying a Configuration

Write Access Rights:

  • Log in to Rhapsody IDE
  • Make changes with the IDE
  • View locker
  • Edit locker
  • Configure check-in comment rules

Rhapsody uses a check out-check in model to lock the engine, message definitions, communication points, and routes for editing.

A component's checked-out or checked-in status is indicated by the presence or absence of a red tick to the left of its icon in the Workspace:

Icon

Description

The communication point is checked in (no tick present).

The communication point is checked out (red tick present).

A selected component is automatically checked out when its Properties dialog is opened and changes are made. Checking in the changes sends a copy of the changed configuration to the Rhapsody engine. Immediately after the check-in, Rhapsody begins processing messages in accordance with the new configuration.

Check-In and Check-Out Actions

Routes and communication points can be checked in and out from their context menu in the Workspace. Alternatively, the check out-check in Toolbar icons can be used to perform the associated tasks:

Icon

 Action

Description

Check In All

Checks in all routes and communication points.
Check Out Selected

Checks out the currently selected route or communication point. This action locks the selected component for editing by the current user only; other users will be able to view but not change its configuration until the component is checked in again.

Checking out a selected route or communication point does not stop it. This means that Rhapsody is able to continue message processing using the 'old' configuration until any changes are checked in.

Check In Selected

Checks in the currently selected route or communication point. This action briefly stops and restarts the selected component.

Undo Check Out

Cancels a check out of the selected component or components and reverts the configuration back to its original state. This overwrites the current configuration and discards any changes that have been made. This action must be confirmed. Following an Undo Check Out, all previously checked out items show as checked in.
Configuration History Launches the Configuration History window to view the configuration history and roll back to earlier versions.
