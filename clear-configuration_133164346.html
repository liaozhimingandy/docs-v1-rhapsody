clear-configuration

The <clear-configuration/> command clears your configuration in a similar manner to selecting the Overwrite the current configuration checkbox in the Reload Local Configuration Wizard. Refer to Loading a Configuration for details.

The <clear-configuration/> command removes:

  • Lockers
  • Folders
  • Routes
  • Communication points
  • Watchlists
  • Web services
  • Lookup tables
  • Tracking schemes
  • Definition files

It does not remove users and groups, Rhapsody variables and certificates. It is intended to be used along with the <rlc-load .../> command to erase the current configuration and load an RLC.

Check-in Comment

checkInMessage is an optional attribute. It can be set to a non-empty string that contains the commit comment to use for the configuration changes. If it is not provided, then an empty commit comment is used.

attribute='value'

Description

 
checkInMessage='<commit_comment>'

Add a commit comment to the RLC.

Example

<!-- Clear the configuration (with optional commit comment) -->
<clear-configuration checkInMessage='Cleared the configuration prior to an RLC load.'/>
