Components Web Service

The Rhapsody Components Web service provides operations to find the communication points and routes in the Rhapsody configuration, simple statistical information about these components, and allows them to be stopped and started.

All these operations use WS-Security to verify that the caller has sufficient privileges. The Web service supports WS-Security with UsernameToken authentication. The password must be provided as plain text.

Once a user has been authenticated, the Web service verifies that they have permissions to make Web service calls and perform the requested operation. A SOAP fault is raised if the authentication or permission check fails.

WSDL Location

https://<RhapsodyEngine>:<WSPort>/services/RhapsodyComponentsService?wsdl

SOAP 1.1 Endpoint

https://<RhapsodyEngine>:<WSPort>/services/RhapsodyComponentsService.RhapsodyComponentsServiceHttpsSoap11Endpoint

SOAP 1.2 Endpoint

https://<RhapsodyEngine>:<WSPort>/services/RhapsodyComponentsService.RhapsodyComponentsServiceHttpsSoap12Endpoint

Operations

getAllComponents

Retrieves an object that represents the folder hierarchy in the Rhapsody configuration, including all communication points, routes, and Web services (which are all in the root folder). Only lockers that the user has view permissions for will be returned. The name, internal ID and current state of each communication point and route is returned.

   <soap:Header>
      <wsse:Security xmlns:wsse="http://docs.oasis-open.org/wss/2004/01/oasis-200401-wss-wssecurity-secext-1.0.xsd">

Retrieves some simple statistical information about a specific communication point, route or Web service. The operation takes two parameters — the ID or full path of the component, and the component type. SOAP faults are raised if the component cannot be found or the user has insufficient privileges to view the component.

startComponent

Starts the specified communication point or route. The operation takes two parameters — the ID or full path of the component, and the component type. SOAP faults are raised if the specified component cannot be found, if an error occurs while starting the component (but not if it starts, then stops shortly afterwards with a connection error), or if the user has insufficient privileges to start the component. This operation has no effect if the component is already running.

SOAP faults are raised if the specified component cannot be found, if an error occurs while stopping the component, or if the user has insufficient privileges to restart the component. For communication points and routes, this operation has the same effect as stopping and then starting them again. For Web services, this causes their connectors to be rebuilt.

SOAP Faults

The Rhapsody Components service raises SOAP faults when syntactical or semantical errors are discovered in the request, or if an error occurs when generating the response. If the request reaches the Rhapsody Components service (that is, it isn't rejected by the framework due to bad XML/SOAP formatting or invalid security parameters), then the SOAP fault returned will include an instance of a ComponentException (defined in the WSDL), which includes an error code and description indicating the problem.

The SOAP fault code will be set to Sender/Client (for SOAP 1.2/1.1) for errors in the request, or Receiver/Server (1.2/1.1) for errors generating the response. The ComponentException object will be included in the details section of the fault. If using SOAP 1.2, then the subcode of the fault will be an instance of componentErrorCode (defined in the WSDL as an enumeration) indicating the error that occurred.

Sample SOAP 1.2 Fault

<soapenv:Envelope xmlns:soapenv="http://www.w3.org/2003/05/soap-envelope">
   <soapenv:Body>
      <soapenv:Fault>
         <soapenv:Code>
            <soapenv:Value>soapenv:Sender</soapenv:Value>
            <soapenv:Subcode>
               <soapenv:Value xmlns:ns1="http://www.orionhealth.com/rhapsody/api/2009/09/rhapsodyComponentsService">ns1:ComponentNotFound</soapenv:Value>
            </soapenv:Subcode>
         </soapenv:Code>
         <soapenv:Reason>
            <soapenv:Text xml:lang="en-US">Component not found</soapenv:Text>
         </soapenv:Reason>
         <soapenv:Detail>
            <ns1:ComponentException xmlns:ns1="http://www.orionhealth.com/rhapsody/api/2009/09/rhapsodyComponentsService">
               <ns1:ErrorCode>ComponentNotFound</ns1:ErrorCode>
               <ns1:Message>Component not found</ns1:Message>
            </ns1:ComponentException>
         </soapenv:Detail>
      </soapenv:Fault>
   </soapenv:Body>
</soapenv:Envelope>
