Many-to-one Mapping Using Mapper Filter

The following steps outline how you can map multiple input messages into a single message:

  1. Download and unzip the ManytoOneTestMapping.zip.
  2. Load the ManytoOneTestMapping.rlc file onto your engine.
  3. In the Directory communication point, ensure the property is set to a valid input directory on your system.
  4. Start all components.
  5. Copy the base message HL7v2_3_ADTA01-OrionBase.edi to the input directory.
  6. When the base message has been processed by the Directory communication point, copy the remaining messages into the input directory.
  7. Launch the Management Console, and verify an output message is in the Hold Queue.
  8. The output message should contain the base message and the OBX repeats from each of the other messages.
