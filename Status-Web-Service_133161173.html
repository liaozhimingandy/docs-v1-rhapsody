Status Web Service

The Rhapsody Status service provides some simple operations to determine the status of the Rhapsody engine. This is limited to simple information that can be obtained without logging on to the Management Console or Rhapsody IDE. Primarily, this is status information while Rhapsody is starting up, and a ping operation to verify that it is still responding. No authentication is required to access any of the operations in this service.

WSDL Location

https://<RhapsodyEngine>:<WSPort>/services/RhapsodyStatusService?wsdl

SOAP 1.1 Endpoint

https://<RhapsodyEngine>:<WSPort>/services/RhapsodyStatusService.RhapsodyStatusServiceHttpsSoap11Endpoint

SOAP 1.2 Endpoint

https://<RhapsodyEngine>:<WSPort>/services/RhapsodyStatusService.RhapsodyStatusServiceHttpsSoap12Endpoint

Operations

ping

Sends a ping request to the Rhapsody engine, which will be responded to if this service is running. Note that a successful ping in no way guarantees messages are actually flowing through Rhapsody. For further information, one of the other services should be used to check for message flow, idle times, and backlogs.

This operation takes no parameters and has no return value.

Sample Request

<soap:Envelope xmlns:soap="http://www.w3.org/2003/05/soap-envelope" xmlns:rhap="http://www.orionhealth.com/rhapsody/api/2009/09/rhapsodyStatusService">
   <soap:Body>
      <rhap:ping/>
   </soap:Body>
</soap:Envelope>

 

Sample Response

<soapenv:Envelope xmlns:soapenv="http://www.w3.org/2003/05/soap-envelope">
   <soapenv:Body>
      <ns1:pingResponse xmlns:ns1="http://www.orionhealth.com/rhapsody/api/2009/09/rhapsodyStatusService"/>
   </soapenv:Body>
</soapenv:Envelope>

getStatus

Requests the current status of the engine and its main internal services. This provides the same information that is available on the Rhapsody Engine Starting/Shutting Down... page of the Rhapsody Management Console.

This operation takes no parameters and returns an object with the current state. For example, activating, deactivating, running, and whether or not some of the internal Rhapsody services are running.

Sample Request

<soap:Envelope xmlns:soap="http://www.w3.org/2003/05/soap-envelope" xmlns:rhap="http://www.orionhealth.com/rhapsody/api/2009/09/rhapsodyStatusService">
   <soap:Header/>
   <soap:Body>
      <rhap:getStatus/>
   </soap:Body>
</soap:Envelope>
