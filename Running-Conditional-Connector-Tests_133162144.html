Running Conditional Connector Tests

Tests can be run using the current configuration of the route, even if the configuration has not been checked in (however, the associated route must be checked in).

To run tests:

  1. On the Conditional Connector Testing dialog, under Settings, configure how the test should be run:

    Field

    Description

    Timeout

    The timeout prevents a badly-configured test from running indefinitely. It defaults to 60 seconds, with a minimum value of 5 and a maximum of 3600 seconds (1 hour).

    Logging Level

    Sets the logging level for the filter while it is being tested. The default is Debug.

  2. To run multiple tests, check the tests you want to run and select the Run Checked Tests button; or to run a single test, select the test and select the Run Test button.

  3. Select the relevant test result link to examine the outcome of a test result in the Conditional Connector Test Results dialog. Refer to Reviewing Conditional Connector Test Results for details.
