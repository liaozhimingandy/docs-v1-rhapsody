Reloading a Configuration

Write Access Rights:

  • Log in to Rhapsody IDE

Reloading the server configuration is not normally necessary. There may be instances, however, when the engine and Rhapsody IDE are not in sync, for example, if you attempt to stop a route or communication point and the engine returns the error: Cannot stop a Route (or Communication point) that is already stopped. Reloading the Rhapsody configuration often corrects this error.

To reload a Rhapsody configuration:

  1. Navigate to Rhapsody>Reload Rhapsody Configuration. The following warning message is displayed:

  2. Select the Yes button to confirm you want to reload the configuration.

    Reloading the current configuration overwrites any recent changes you have made and have not checked in.
