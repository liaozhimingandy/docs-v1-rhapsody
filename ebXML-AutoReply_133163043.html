ebXML AutoReply

The ebXML AutoReply filter is a convenience filter which encompasses the response generating functionality of the ebXML Acknowledgment filter, the ebXML Ping Service filter and the ebXML Message Status filter. Typically, these three filters will be used only to respond to incoming ebXML messages, rather than generate new, outgoing ebXML messages. In this common case, rather than using all three filters, it is convenient to use this single AutoReply Filter to provide the functionality while keeping the route uncomplicated.

Configuration of this filter is limited to deciding whether to sign any of the three generated message types (note that if signing of acknowledgment messages is enabled, it will occur only if the original message requested a signed acknowledgment).

The filter itself will determine which type of message to generate, based on the input message, and that if an input message is not an acknowledgment request, a Ping message or a StatusRequest message, no processing will occur on that message and it will be output unchanged.

It is recommended that, as a best practice, all receiving routes implementing ebXML functionality include either the ebXML AutoReply filter or each of the ebXML Acknowledgment filter, ebXML Ping Service filter and ebXML Message Status Service filter. Refer to ebXML Filters for details.

Configuration Properties

Property

Description

Acknowledgment Message Signing

Whether the filter should sign generated acknowledgment messages if the sender of the original message has requested it:

  • Do Not Sign Messages (default).
  • As Per Message Property.
  • Sign Messages.

Pong Message Signing

Whether the filter should sign generated Pong messages:

  • Do Not Sign Messages (default).
  • As Per Message Property.
  • Sign Messages.

StatusResponse Message Signing

Whether the filter should sign generated StatusResponse messages.

  • Do Not Sign Messages (default).
  • As Per Message Property.
  • Sign Messages.

Signature Algorithm

The algorithm to use to sign messages. Currently RSA with SHA-1 and DSA with SHA-1 are supported. The alternative message property is eb:SignatureAlgorithm.

  • rsa-sha1 (default).
  • dsa-sha1.

Signing Key

The private key to use to sign messages. The alternative message property is eb:SignatureKeyAlias whose value must be the alias of the private key to use.
