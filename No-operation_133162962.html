No-operation

The No-operation (pass-through) filter passes all messages (unchanged) from input to output. This filter is typically used to provide a point of connection for route branches or for connecting an input communication point to a conditional connector.

This is also useful to use for route to route delivery. It provides a 'de-coupling' of the routes, allowing one route to change without the other having to know about the changes.

There are no configurable properties for this filter.

Rhapsody does not extract route properties if two communication points are linked together without any filters. Route properties are only extracted when the messages reach the filters. This may cause problems in the following situations:

  • When the Management Console displays or searches for extracted properties.
  • If the output communication point uses the extracted properties as part of its configuration. For example, a directory communication point may use an extracted property as the base output file name.

To force messages to extract route properties on a pass-through route, add a No-operation filter between the connected communication points.
