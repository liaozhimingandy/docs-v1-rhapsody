Search and Replace

The Search and Replace filter performs simple text substitution on messages being processed; for example, parsing an XML type message, then searching within the message for an <ADDRESS> XML tag and replacing it with a <patient_address> tag.

If replacement text is specified, this filter performs a true search and replace. If no replacement text is specified, any and all instances of the search text are removed from the message.

The Search and Replace filter performs matching based on the bytes of the strings rather than characters. This can result in false matching, for example, the character Є (04 04 in UTF-16) is found in the message body ԄЅ (05 04 04 05 in UTF-16). The chances of this problem occurring are minimal, especially if the search string is longer than one character. However, if you believe you are likely to encounter this issue, then you can avoid it by using the JavaScript filter and the string replace method, for example:

var next = output.append(input[0]);
next.text = next.text.replace('Є', 'ԄЅ')

The Search and Replace filter does not support regular expressions.

Configuration Properties

Property

Description

Search String

The text to search for within the message.

Replace String

The text with which to replace the search string. This may be empty to delete all instances of the search string from the message.

Encoding

The encoding of the search and replace strings.

The search and replace parameters are case sensitive.

When performing a search and replace on an XML message, both the opening and closing XML message tags must be replaced; therefore, the search only looks for the closing parenthesis on the XML tags.
