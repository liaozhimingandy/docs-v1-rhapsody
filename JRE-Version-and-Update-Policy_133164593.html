JRE Version and Update Policy

This section describes the bundled versions and standard Rhapsody policy for Java Runtime Environment (JRE) updates:

  • JRE Version Policy
  • Selecting the JRE Version
  • JRE Versions

JRE Version Policy

Rhapsody's JRE version policy is as follows:

  • Every new version of Rhapsody includes a recent stable version of the JRE.
  • An update to the JRE version is typically performed with every General Availability (GA) release of Rhapsody, subject to an updated version being available since the previous GA release.
  • Due to inherent stability risks, Rhapsody does not provide JRE updates for previous releases of Rhapsody.

Selecting the JRE Version

Rhapsody as an organization:

  • Favors a stable release that has mainstream acceptability in terms of maturity and inclusion of fixes for security vulnerabilities and other bug fixes.
  • Performs a risk assessment of the JRE update and its effects on Rhapsody.

When an upgrade to the latest version of Rhapsody (to obtain the latest security and bug fixes) is not an option, Rhapsody as an organization recommends you refer to the Java vendor's website for possible mitigation actions.
