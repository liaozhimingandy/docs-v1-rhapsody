Watchlist Configuration

Watchlists provide a method for logically grouping components together and providing a visual indicator of any alerts occurring on one of the components. They also provide a scheduling mechanism that allows the redirection of notifications to different users based on their defined shifts. The status of all watchlists is displayed on the Engine Monitor page of the Management Console.

Creating Watchlists

To create a watchlist:

  1. In the Management Console, navigate to Notifications>Watchlists.

  2. Enter the name of the watchlist in the text box and click the Add icon .

    Watchlists names are case-sensitive and no two watchlists can have the same name.

Adding Components to a Watchlist

By far, the best way to add Components to a Watchlist is from the Monitoring pages.

  1. In the Management Console, navigate to Monitoring>Communication Points.
  2. Select the communication points that need to be added to the Watchlist.
  3. Click the Add Notifications link to display the Add Notification dialog.

  4. Select Watchlist from the Send To drop-down list and enter/search for the Watchlist name.

    Entering a space in the search box will show the first ten Watchlists.

  5. Click the Add icon and review the Resend and Escalation Periods as appropriate.

  6. Select the Apply button to save the changes made to the page.

    The same steps apply to adding Routes as a Watchlist.

You can also add components to a Watchlist from the Watchlist Details page:

  1. In the Management Console, navigate to Notifications>Watchlists.
  2. Select the Watchlist you wish to modify from the list.
    1. Use the Add Component or Add Web Service text boxes to search for a component by name OR
    2. Select the Folder icon and select components from the Select Component dialog.

Subscribing Users to a Watchlist

  1. In the Management Console, navigate to Notifications>Watchlists.
  2. Select a Watchlist from the list.
  3. In the People To Notify panel, enter a name in the text box and then click the Add icon .

  4. If you have shifts defined for your watchlist, use the checkboxes beside the added user to choose during which shifts, this user should receive notifications for this Watchlist. If you have no Shifts defined, the user will receive notifications from this Watchlist 24/7.

    You can confirm the chosen user has Notification Addresses set by clicking on their name in the People To Notify list.

  5. Select the Apply button to save the changes made to the page.

Creating a Schedule for a Watchlist

  1. Navigate to Notifications>Watchlists and select a Watchlist from the list.
  2. Click the Create a Schedule link in the People To Notify section.

  3. Under the Shifts section, enter a From time in the textbox for the Daytime Shift 1 or After Hours.

    The shifts starting time must be in an ascending order otherwise notifications will be sent to wrong recipients.

    Valid Shifts

    Invalid Shifts

    Daytime Shift 1 : 06:00 - 14:00
    Daytime Shift 2 : 14:00 - 22:00
    After hours Shift 3 : 22:00 - 06:00

    Daytime Shift 1 : 09:00 - 17:00
    Daytime Shift 2 : 17:00 - 01:00
    After hours Shift 3 : 01:00 - 08:00
  4. Select the Apply button to save the changes.

The best way to group and then split notifications between teams is through Watchlists. You can create a Watchlist per team and then subscribe the team members to the Watchlist.
