Using Rhapsody IDE for the First Time

To log on to Rhapsody IDE for the first time:

  1. Ensure the Rhapsody service daemon is running.

  2. Launch Rhapsody IDE. The Rhapsody Login dialog is displayed:

  3. Enter the username administrator and the Password rhapsody (which is case-sensitive). This password can only be used once. After your first successful login, you are prompted to change it.

  4. Select the engine you want to connect to. If the engine is on another machine, select the Browse... button and navigate to the appropriate installation. By default, Rhapsody attempts to link to the local installation of the interface engine.

    The Browse for Rhapsody Engines dialog only includes those engines that are compatible with the current version of Rhapsody IDE.

    The Windows速 firewall does not allow UDP broadcast messages by default. As this is the mechanism used by Rhapsody IDE to search for available Rhapsody engines, no engines may be listed in this dialog. This issue can be resolved by adding Rhapsody IDE to the Windows速 Firewall list of exclusions.

  5. Select the More Options button if you want to:

    • View or edit the Rhapsody Port or the Broadcast Port.

    • Access the Rhapsody IDE Trust Manager by clicking the Rhapsody IDE Trust Manager link.

    • Reset the list of engines you have connected to in the Rhapsody Engine field to localhost by clicking the Clear Rhapsody Engine List link.

  6. Select the OK button to log onto Rhapsody IDE.

Unable to Connect

If you are unable to connect to the specified Rhapsody engine the following error dialog may be displayed:

To troubleshoot this, perform the following actions:

  1. Ensure the Rhapsody service is running.
    • On Windows速:
      1. Navigate to Start>Control Panel>Administrative Tools>Services. This path may differ slightly depending on the version of Windows速 you are using.
      2. Right-click the Rhapsody service, then select Start.

    • On *nix, enter the command:

      ./rhapsody.sh start
  2. Check the Port details.
    • If the Rhapsody engine is not using the default ports, ensure that the correct ports are entered on the Rhapsody Login dialog.
