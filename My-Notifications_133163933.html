My Notifications

Menu Path : Notifications>My Notifications

Write Access Rights:

  • Log in to Management Console
  • Edit current user's notification preferences
  • Send test notifications
  • Edit other users' notification preferences

A notification is a set of identified trigger events that can be associated with one or more Rhapsody routes or communication points. For a list of notifications available in Rhapsody, refer to Notifications Reference.

The My Notifications page enables the user to configure the email address, SMS phone number, or pager number to which notifications are sent.

Refer to Delivery Methods for details on how to configure delivery methods (email, SMS or pager) to send notifications.

To avoid an overload of notifications after engine start-up, notifications are suppressed for the first five minutes after the engine is started. You are notified of when notifications have been enabled through the System Monitor activity feed with the log entry: Starting the notification resender.

If a user has the 'Edit other users' notification preferences' access right, they can configure notifications for other users. To access notification settings for other users:

  1. Navigate to the Watchlists page.
  2. Select a user from the People to Notify table or from the Upcoming Leave table.

Refer to the following pages for details:

  • Send Notifications To
  • My Leave (Do not send me notifications)
  • My Notifications

3-Step Guide For Setting up Notifications

The 3-step guide appears at the top of the My Notifications page for all users when an administrator or developer has navigated there by clicking on the page link in the 3-step guide. Refer to Three-step Guide for Setting up Notifications for more details.
